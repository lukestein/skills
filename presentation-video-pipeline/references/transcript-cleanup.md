# Transcript Cleanup

Use this standard when the user wants a readable Markdown transcript rather than raw subtitles.

## Goals

- Preserve meaning.
- Preserve the speaker's distinctive tone.
- Preserve all specific critiques, claims, examples, and caveats.
- Improve readability without turning the transcript into a summary.

## Always Do

- Correct obvious terminology mistakes when context makes the intended term clear.
- Remove filler words such as `um`, `uh`, and simple stutters when they add no meaning.
- Break long transcript walls into readable paragraphs.
- Add an H1 for the talk title.
- Add H2 headings for real topic shifts, not every minor tangent.

## Do Not Do

- Do not summarize arguments.
- Do not soften or remove criticism.
- Do not delete uncertainty language when the speaker is being tentative.
- Do not over-edit into a different voice.

## Good Topic Shift Headings

- `## Motivation`
- `## Data and Measurement`
- `## Identification Strategy`
- `## Results`
- `## Questions From The Audience`

The actual headings should come from the content, not from this list.

## Terminology Strategy

When the domain is technical, build a small glossary before the final cleanup pass:

- speaker names
- coauthor names
- institution names
- cited papers
- software and package names
- domain terms appearing on slides or in the video title

If you later add OCR for slide text, use that OCR output as another source for the glossary.
